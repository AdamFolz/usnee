# RFC Decisions — USNEE Project

## RFC-1: Экономика XP
**Решение:** Оставить текущую модель (10 XP за инъекцию).  
**Обоснование:** Без введения дополнительных ограничений или синков. Балансировка переносится на этап продуктовой аналитики после релиза.

## RFC-2: Защита от abuse
**Решение:** Оставить текущую server-side валидацию как достаточную на данном этапе.  
**Обоснование:** Дополнительные антифрод-слои (rate limit / fingerprint / heuristics) отложены. Преждевременная оптимизация до появления реального трафика.

## RFC-3: Масштабирование
**Решение:** Redis / CDN / архивация не внедряются сейчас.  
**Обоснование:** Текущая архитектура признаётся достаточной для MVP и раннего роста. Рефакторинг под нагрузку — только по метрикам продакшена.

## Статус архитектурных фаз

| Фаза | Статус | Примечание |
|------|--------|------------|
| Backend API + Database | ✅ ЗАКРЫТА | Изменения только по багам или метрикам реального продакшена |
| Frontend (React + TS + Tailwind) | 🔄 В РАБОТЕ | Текущий этап |
| Telegram Bot | ⏳ ОЖИДАЕТ | Следующий после Frontend |
| DevOps / Deploy | ⏳ ОЖИДАЕТ | После завершения Frontend и Bot |

## Дата фиксации
RFC приняты и зафиксированы на моменте перехода к Frontend.
