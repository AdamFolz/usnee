# Установка USNEE на Windows (с Docker)

Docker — это программа, которая запускает все части проекта (backend, frontend, базу данных, бота) в отдельных "контейнерах". Тебе не нужно ставить Python, Node.js, PostgreSQL отдельно — Docker всё сделает сам.

---

## Шаг 1. Установить Docker Desktop

1. Открой браузер и перейди на сайт: **https://www.docker.com/products/docker-desktop**
2. Нажми кнопку **"Download for Windows"** (бесплатная версия)
3. Скачай установщик и запусти его
4. Жми **"Next"** → **"Install"** → дождись окончания
5. **Перезагрузи компьютер**
6. После перезагрузки открой программу **Docker Desktop** (должна запуститься сама или найди в меню Пуск)
7. Пропусти регистрацию (можно "Skip" или "Continue without signing in")
8. Дождись, пока внизу окна появится зелёная надпись **"Docker Engine running"**

Проверка: открой **Командную строку** (Win+R → `cmd` → Enter) и введи:
```
docker --version
```
Если вышло что-то вроде `Docker version 24.x.x` — всё ок.

---

## Шаг 2. Настроить .env (настройки проекта)

1. Открой папку проекта: `M:\SEKS\АКТУАЛЬНОЕ\USNEE\`
2. Найди файл **`.env`** (если не видишь — включи показ скрытых файлов)
3. Открой его **Блокнотом** (Notepad)
4. Замени `your_bot_token_here` на реальный токен от Telegram-бота (получить у @BotFather)
5. **Сохрани файл** (Ctrl+S) и закрой

Если пока нет токена — можешь оставить как есть, но бот работать не будет. Приложение (Mini App) всё равно запустится через браузер.

---

## Шаг 3. Запустить проект

1. Открой **Командную строку** (Win+R → `cmd` → Enter)
2. Перейди в папку проекта:
```
cd M:\SEKS\АКТУАЛЬНОЕ\USNEE
```
3. Запусти все контейнеры:
```
docker-compose up -d
```

(Жди, пока скачаются и соберутся образы — первый раз это может занять 5-10 минут. Потом будет быстрее.)

4. Проверь, что всё работает:
```
docker-compose ps
```

Должно показать 5 строк (postgres, redis, backend, bot, nginx) — все со статусом **"Up"**.

---

## Шаг 4. Открыть приложение

Открой браузер и перейди по адресу:

```
http://localhost
```

Если всё ок — откроется USNEE Mini App.

API доступен по адресу:
```
http://localhost/api/health
```
(должен вернуть `{"status": "ok"}`)

## Как остановить

В командной строке (в папке проекта):
```
docker-compose down
```

Чтобы остановить и удалить все данные (базу данных):
```
docker-compose down -v
```

---

## Как перезапустить

```
docker-compose down
docker-compose up -d
```

---

## Что делать, если что-то не работает

| Проблема | Решение |
|----------|---------|
| `docker-compose` не найден | В новых версиях Docker используй `docker compose` (без дефиса) вместо `docker-compose` |
| Порт 80 занят | Закрой программу, которая использует порт 80 (например, Apache, IIS, Skype) или измени порт в `docker-compose.yml` |
| Контейнеры не запускаются | Введи `docker-compose logs` и скопируй ошибку |
| База данных не создалась | Введи `docker-compose down -v` и снова `docker-compose up -d` |
| Frontend не обновляется | После изменений кода пересобери: `docker-compose build frontend && docker-compose up -d` |
