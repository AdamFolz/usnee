# USNEE (ЮЗНИ) — Roadmap до финальной версии

## Философия

**Без осуждения, только факты.**  
Пользователь решает сам, что и как. Мы даём инструменты, не оценки. Ироничные ачивки — про ситуацию, не про человека.

---

## Фаза 1: MVP — Инъекции (текущий фокус)

**Статус:** 🔄 В работе

| Модуль | Задачи |
|--------|--------|
| **Frontend** | Форма записи (метод, место, объём, триггер), статистика, история, ачивки, экстренная помощь |
| **Backend** | FastAPI + PostgreSQL, Telegram WebApp auth, event-driven ачивки, cursor-based пагинация |
| **Bot** | /start, WebApp кнопка, /help, быстрая статистика |
| **DevOps** | Docker Compose, nginx, CI/CD скелет |

**Критерий завершения:** Приложение работает в Telegram Mini App, пользователь может записать инъекцию и увидеть статистику.

## Фаза 5: Расширение способов употребления

**Статус:** ⏳ Ожидает

| Способ | Код | Что фиксируется |
|--------|-----|-----------------|
| **Инъекция** | `injection` | Метод, место, объём, партия |
| **Курение** | `smoking` | Устройство (фольга, pipe, бонг), доза, смесь |
| **Принятие внутрь** | `oral` | Доза, пустой/полный желудок, время до эффекта |
| **Нюхание** | `snorting` | Поверхность, доза, разбавление |
| **Вдыхание** | `inhalation` | Газ/растворитель, объём, вентиляция |
| **Ректальное** | `rectal` | Доза, растворитель |
| **Трансдермальное** | `transdermal` | Пластырь, площадь, время контакта |

**Архитектура:** Таблица `consumption_events` с полиморфным типом (discriminator column) или JSONB полем `details`. Унифицированный API для всех способов.

## Фаза 6: Классификация ПАВ

**Статус:** ⏳ Ожидает

| Класс | Примеры | Особенности отслеживания |
|-------|---------|--------------------------|
| **Опиоиды** | Героин, метадон, бупренорфин | Интервалы, симптомы отмены, партии |
| **Стимуляторы** | Метамфетамин, амфетамин, кокаин | Сон, питание, «допинг-циклы» |
| **Диссоциативы** | Кетамин, МКС, ПКП | Качество, толерантность, курс лечения |
| **Каннабиноиды** | Марихуана, синтетика | Способ (курение/еда), % THC, симптомы |
| **Алкоголь** | — | Стандартные единицы, BAC оценка |
| **Бензодиазепины** | — | Потенциал отмены, длительность |
| **Пользовательский** | Любой | Свои названия, свои параметры |

**ПАВ-агностика:** Пользователь может создать свой ПАВ с произвольными параметрами.

## Архитектурные решения для будущего расширения

```
consumption_events
├── id, user_id, created_at
├── substance_type (enum: opioid, stimulant, dissociative, cannabinoid, alcohol, benzo, custom)
├── substance_name (varchar, кастомное название)
├── consumption_method (enum: injection, smoking, oral, snorting, inhalation, rectal, transdermal)
├── details (JSONB) — полиморфные поля в зависимости от метода
│   ├── injection: {method, site, volume_ml, batch_id}
│   ├── smoking: {device, dose_mg, mixture}
│   ├── oral: {dose_mg, stomach_status, onset_minutes}
│   └── ...
├── trigger, trigger_note
├── xp_reward (int, default 10)
├── is_cancelled
└── ...
```

**Таким образом, MVP (инъекции) — это просто частный случай `consumption_method = 'injection'`.** Вся текущая архитектура уже готовится к расширению.
